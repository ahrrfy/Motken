# قواعد إلزامية — نظام مُتْقِن

## 🚨 البروتوكول الأعلى: كل جلسة مستقلة

### ⛔ محظورات مطلقة — لا استثناء:
1. **لا تقرأ ولا تعتمد على أي ملف تقارير أو سجلات من جلسات سابقة** — إذا وجدت ملفاً اسمه fix-log أو maintenance-log أو report أو أي سجل مشابه → **تجاهله تماماً وافحص الكود بنفسك**
2. **لا تُنشئ ملفات تقارير أو سجلات أو fix-log أو فحوصات** — لا ملف واحد. كل شيء يبقى في المحادثة فقط
3. **لا تقل "حسب الإصلاحات السابقة" أو "وفقاً للتقرير"** — لا يوجد شيء اسمه "سابق". افحص الكود الآن بعينك
4. **لا تفترض أن شيئاً يعمل لأن جلسة سابقة قالت ذلك** — تحقق بنفسك أو قل "لم أتحقق بعد"

### 🧹 عند نهاية الجلسة:
- **احذف أي ملف أنشأته خلال الجلسة** من نوع: تقارير، سجلات، ملفات فحص، ملفات معلومات إصلاح
- **لا تترك أي ملف** يمكن لوكيل لاحق أن يعتمد عليه بدلاً من فحص الكود الحقيقي
- الملفات المسموح إنشاؤها فقط: **كود المشروع نفسه** (مكونات، مسارات، اختبارات، إعدادات)

## ✅ قواعد التحقق من الإصلاحات (داخل الجلسة فقط)

### عند إنهاء أي إصلاح:
1. **تحقق بـ grep** أن الكود الجديد موجود فعلاً في الملف
2. **شغّل `npx vite build --mode development`** وتأكد من نجاحه
3. **لا تقل "تم الإصلاح"** إلا بعد الخطوتين أعلاه
4. أعرض النتيجة للمستخدم مباشرة — **لا تسجّلها في ملف**

### قاعدة الـ 3 محاولات:
- إذا نفس الإصلاح فشل 3 مرات ← **توقف فوراً واسأل المستخدم**
- لا تكرر نفس المنهج — إذا فشل مرتين، **غيّر الطريقة بالكامل**

### قاعدة الصدق:
- ❌ لا تقل "مكتمل" إذا لم تتحقق بـ grep + build
- ❌ لا تقل "يعمل" إذا لم تختبر فعلياً
- ❌ لا تقل "لا توجد مشاكل" بدون فحص حقيقي
- ✅ إذا لا تعرف ← قل **"لم أتحقق بعد"**
- ✅ إذا فشلت ← قل **"فشلت والسبب هو X"**

## 📁 بنية المشروع
- **Frontend**: React 19 + Vite 7 + TypeScript + Tailwind 4 + Shadcn/UI (`client/`)
- **Backend**: Express 5 + Drizzle ORM + PostgreSQL (`server/`)
- **Schema**: `shared/schema/` (11 وحدة)
- **Storage**: `server/storage/` (12 وحدة)
- **Tests**: Vitest (`server/__tests__/`)
- **Port**: 5002
- **اللغة**: عربي (RTL) — خط Cairo
- **Logger**: Pino (`server/lib/logger.ts`)
- **Deploy**: Docker + GitHub Actions
